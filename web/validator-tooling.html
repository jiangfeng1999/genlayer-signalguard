SignalGuard Validator Ops Probe

A lightweight operations checklist for validator health, metrics, balance, and snapshot readiness.

Tooling for running a Validator Observed cap 1,000 BP No credentials required for review

Probe Matrix

Endpoint Purpose Alert rule
/health Check node, database, and consensus health before treating a validator as ready. Alert when status is not up or a required check reports down.
/metrics Expose Prometheus metrics for node, GenVM, and WebDriver resource tracking. Alert on missing scrape data, high CPU, memory pressure, or stalled block progress.
/balance Track account balance needed for operational transactions. Alert when balance drops below the operator-defined threshold.
/snapshot Create an operator-authorized state snapshot for maintenance and upgrades. Require an operator token and local or tunneled access before invocation.

Boundaries

  • This is operator tooling; it does not claim the current wallet is running a validator.
  • Snapshot handling is protected because it requires an operator token.
  • Reviewers can adapt the fixture to local, tunneled, or hosted validator endpoints.
