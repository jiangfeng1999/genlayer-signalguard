SignalGuard Gas Fee Simulator Tests

A deterministic test matrix for fee-aware SignalGuard read and write flows.

Gas Fees Simulator Tests Observed cap 1,000 BP Studio limitation documented

Purpose

The tests separate read-only calls, state-changing writes, malformed inputs, and repeated writes. The package is explicit that Studio is useful for functional prototyping but does not consume gas, so live clients still need fee estimation and user confirmation.

Cases

Case Expected fee behavior Risk controlled
Read latest verdict View calls should avoid write transactions and wallet fee prompts. Accidental write flow for read-only state.
Write review claim State-changing calls require signing, processing, and sufficient balance. Studio-only testing hides live fee and finalization behavior.
Oversized source URL Client-side validation should reject malformed inputs before writes. Bad input can waste fees and user time.
Repeated review claim Repeated writes should be treated as separate fee-bearing transactions. Prototype interactions can make repeated writes look free.
