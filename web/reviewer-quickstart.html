SignalGuard Reviewer Quickstart

A five-minute path through the live application, deployment, source, and automated evidence.

Live review

1. Open the SignalGuard DApp. The page shows the latest verified deployment snapshot without requesting wallet access.

2. Inspect the write path. `web/signalguard-dapp.js` connects an injected wallet, switches to StudioNet, calls review_claim, waits for an accepted receipt, and refreshes state.

3. Inspect the deployed contract. The public schema has no constructor arguments and exposes three expected methods.

Reproduce the deployment proof

Clone the repository, use Python 3.12, and run:

python -m pip install -r requirements-dev.txt
python scripts/verify_studionet_deployment.py

The verifier downloads the deployed source, compares normalized SHA-256 values, checks the schema, and performs public state reads. It does not broadcast a transaction.

Run the local checks

python -m pytest tests/direct -q
pwsh ./scripts/verify-evidence-package.ps1

Direct Mode is intended for Linux because the current test loader uses temporary stdin semantics that Windows rejects. GitHub Actions runs the Linux path automatically.

Evidence boundaries

  • The Project claim is based on the DApp, deployed contract, and reproducible verification.
  • Research, tutorials, and dashboards are supporting evidence, not duplicate Project submissions.
  • The repository does not claim mocked write-path coverage for the old `v0.2.16` prompt template.
  • Final scoring and acceptance remain a GenLayer review decision.
