SignalGuard Reviewer Quickstart Documentation

A concise operational guide for reproducing the SignalGuard GenLayer evidence package.

Documentation Reviewer Guide Updated 2026-07-07

Purpose

This page is for reviewers who want to inspect SignalGuard quickly without reading every repository file first. It points to the contract, the static demo, sample inputs, public dashboard, and one-command verifier.

Fast Review Path

1. Inspect the contract. Open contracts/signal_guard.py and confirm the public entry points: review_claim and latest_verdict.

2. Prepare a sample call. Open the static demo or run the CLI helper to generate a review_claim payload.

3. Check source-grounding cases. Use sample_claims.json and adversarial_claims.json to review supported, contradicted, and inconclusive cases.

4. Verify public status tooling. Open the Portal dashboard and compare it with the status script output.

5. Run the package verifier. Clone the repository and run the command below from the repository root.

powershell -ExecutionPolicy Bypass -File scripts\verify-evidence-package.ps1

Boundaries

  • This guide documents how to reproduce the public evidence package; it is not a second Project submission.
  • It does not include private Portal submission status, wallet signatures, or reCAPTCHA state.
  • Final Portal scoring remains a GenLayer review decision.
