SignalGuard Graybox Harness

Practical grayboxing controls for source-grounded GenLayer Intelligent Contracts.

Grayboxing Observed cap 1,000 BP Prompt-control cases

Purpose

The harness turns grayboxing principles into concrete tests for SignalGuard. It separates trusted task instructions from untrusted source text, bounds retrieved source excerpts, restricts the allowed verdicts, and records the risk each case is testing.

The design follows the GenLayer grayboxing model: input filtration, output restrictions, model isolation assumptions, and monitoring of suspicious source or verdict behavior.

Cases

Case Risk Control
Prompt injection in source Retrieved source text asks the model to ignore the application task. Keep trusted task instructions separate and restrict the verdict schema.
Oversized source excerpt Long input dilutes evidence or increases cost. Normalize and bound the source excerpt.
Conflicting claim and source Claim overstates what the cited source proves. Require source-grounded rationale.
Ambiguous reward claim User infers guarantees from incomplete Portal data. Allow inconclusive when evidence does not prove the claim.
