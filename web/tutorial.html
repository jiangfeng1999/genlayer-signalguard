Build a Source-Grounded Claim Review Contract

A practical GenLayer tutorial for fetching a cited source, asking validators for a constrained verdict, and preparing a Studio call payload.

Educational Content GenLayer Intelligent Contracts Updated 2026-07-07

What You Will Build

The goal is a small Intelligent Contract that answers one review question: does a cited public source support a submitted claim? The result is one of supported, contradicted, or inconclusive.

This is useful for ecosystem announcements, research notes, and agent workflows where a claim needs to be checked against human-readable evidence rather than a numeric oracle value.

Prerequisites

  • A GenLayer Studio environment for deploying an Intelligent Contract.
  • A public source URL that reviewers can inspect.
  • A specific claim that can be judged from that source.
  • The SignalGuard repository for reference code and helper scripts.

Implementation Steps

1. Keep contract state small

The first version stores only the latest claim, source URL, and verdict report. This keeps deployment and review simple.

claim: str
source_url: str
verdict_report: str

2. Fetch the cited source inside the decision flow

SignalGuard uses GenLayer web access so the validator task is grounded in the cited source instead of a free-form model answer.

response = gl.nondet.web.get(source_url)
page_text = response.body.decode("utf-8")

3. Ask for a constrained verdict

The task should force one compact verdict and forbid invented facts. A narrow prompt makes the output easier to review.

report = gl.eq_principle.prompt_non_comparative(
    lambda: prompt_input,
    task="Determine whether the claim is supported, contradicted, or inconclusive based only on the source content.",
    criteria="Return compact JSON with verdict, confidence, and rationale. Do not invent facts.",
)

4. Build a Studio call payload

The CLI helper prepares the method and arguments so a reviewer can copy them into Studio.

python app/signalguard_cli.py \
  "GenLayer can use AI validators for subjective decisions" \
  "https://docs.genlayer.com/understand-genlayer-protocol/what-is-genlayer"

Review Checklist

  • The source URL is public and stable enough for review.
  • The claim is specific enough to evaluate.
  • The prompt criteria forbid unsupported facts.
  • The output format is compact and predictable.
  • The accepted validator result is stored for app-side inspection.

Extensions

  • Store verdicts by claim hash instead of keeping only the latest result.
  • Add source freshness metadata.
  • Compare multiple independent sources.
  • Deploy a frontend that reads the latest verdict.
  • Add more supported, contradicted, and inconclusive test cases.
