SignalGuard Adversarial Testing

A compact test suite for checking whether source-grounded verdicts reject unsupported summaries, overclaims, and future-looking speculation.

Adversarial Testing 6 Cases Updated 2026-07-07

Purpose

SignalGuard should only mark a claim as supported when the cited source directly backs it. Reviewers can deploy the contract, run each case, and compare the returned verdict with the expected verdict.

How To Run

  1. Deploy contracts/signal_guard.py in GenLayer Studio.
  2. Open examples/adversarial_claims.json.
  3. Call review_claim(claim, source_url) for each case.
  4. Read latest_verdict().
  5. Compare the returned verdict with expected_verdict.

Pass Criteria

  • Supported claims cite evidence that directly backs the claim.
  • Contradicted claims identify source evidence that conflicts with the claim.
  • Inconclusive claims avoid invented facts and future predictions.
  • Rationales do not cite facts absent from the fetched source.
