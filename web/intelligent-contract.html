SignalGuard Intelligent Contract

A deployed source-grounded verdict primitive for claims that deterministic contracts cannot evaluate.

Intelligent ContractsStudioNetGenVM v0.2.16Source verified

Use case

A caller provides a claim and the public source that should support it. SignalGuard asks GenLayer validators to decide whether the cited evidence supports, contradicts, or cannot resolve that claim. The accepted report becomes shared contract state for applications, agents, or community review workflows.

Consensus flow

1. InputReceive a claim and source URL.
2. EvidenceFetch source content with `gl.nondet.web.get`.
3. DecisionApply explicit verdict criteria through `prompt_non_comparative`.
4. StateStore the accepted claim, source, and report.

Boundaries

  • The current deployment stores the latest review rather than an unbounded history.
  • The report is constrained JSON text; typed verdict fields belong in a future deployment.
  • The DApp enforces HTTPS and bounded inputs, while stronger contract-level URL policy is future work.
  • The old prompt template is not supported by current Direct Mode LLM mocks; the repository states this limitation explicitly.
